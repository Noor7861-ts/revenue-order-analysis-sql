Revenue & Order Behavior Analysis (SQL | BigQuery)
Overview

This project analyzes transactional data to understand key drivers of business performance, focusing on revenue trends and order behavior.

Objectives

- Analyze daily revenue patterns
- Evaluate order volume fluctuations
- Identify whether revenue is driven by order volume or order value

Key Analysis Performed

- Daily revenue and order volume analysis
- Average revenue per order calculation
- Identification of top and low-performing days
- Comparison of order volume vs order value impact

Key Insights

- Revenue is not solely driven by order volume; average order value plays a significant role
- Some high-volume days generated lower revenue due to smaller transaction values
- Higher revenue days were often associated with higher average order values
- Low-performing days were primarily driven by lower order value rather than just reduced order count

Data Limitations

- Profit analysis was limited due to missing or zero cost and shipping data
- Analysis was focused on revenue-based metrics
  
Tools Used
- Google BigQuery (SQL)
